# 🎾 테니스 예약 웹 서비스 만들기 (Vercel + Render 서버 아키텍처)

GitHub Actions 환경에서 크롬(Headless)을 띄우는 것이 불안정하셨군요! 맞습니다. 깃허브 버추얼 머신이나 Vercel의 기본 환경은 셀레니움 같은 "무거운 브라우저 자동화"를 장시간 돌리기 위해 만들어진 것이 아니라서 종종 끊기거나 막히는 문제가 발생합니다.

초보자 분들이 가장 안정적이면서 **무료**로 웹 서비스를 구축하는 추천 아키텍처는 다음과 같습니다. 지난번에 추천받으신 **Vercel + Next.js**의 깔끔한 사용자 화면 유지하면서, 뒤에서 셀레니움을 튼튼하게 실행해 줄 "진짜 서버"를 붙이는 방법입니다.

---

## 🛠️ 추천 아키텍처 (2-Tier 구조)

매크로를 24시간 안정적으로 돌리기 위해서는 역할을 둘로 나누는 것이 정석이자 가장 세련된 방법입니다.

1. **프론트엔드 (리모컨 화면) - Vercel + Next.js**
   - 무료 도메인 제공 (예: `my-tennis.vercel.app`)
   - 사용자가 숫자를 입력하고 실행/중지를 누를 수 있는 눈에 보이는 깔끔한 웹사이트.
2. **백엔드 API 서버 (크롬 봇 실행기) - Render.com (무료 호스팅)**
   - 파이썬 코드(`main.py`)가 24시간 상주하며 돌아갈 진짜 리눅스 서버.
   - Vercel 웹에서 "12분 간격으로 실행해"라는 신호를 이 API 서버로 쏘면, 서버 내부에서 백그라운드로 셀레니움 봇을 가동합니다.

---

## 📝 Step-by-Step 구축 순서

### STEP 1. 백엔드 (Render.com)용 파이썬 API 서버 만들기
기존 `main.py`를 외부에서 웹 신호로 시작/종료할 수 있는 구조(FastAPI)로 감싸야 합니다.

1. **`server.py` 파일 생성 (FastAPI 이용)**
   서버 폴더(예: `backend`)를 하나 만들고, FastAPI 패키지를 이용해 웹 요청을 받아들일 수 있는 짧은 파이썬 코드를 작성합니다.
   ```python
   # server.py 예시 코드
   from fastapi import FastAPI
   import asyncio
   import main # 기존 매크로 파일 임포트

   app = FastAPI()
   bot_task = None # 백그라운드 매크로 작업

   @app.post("/start")
   async def start_bot(interval: int):
       global bot_task
       if not bot_task:
           bot_task = asyncio.create_task(main.run_macro_loop(interval))
       return {"msg": f"{interval}분 단위로 매크로 시작!"}
   ```
2. **Render 호스팅 배포**
   - [Render.com](https://render.com/) 에 접속해 GitHub 저장소를 연동합니다.
   - Web Service를 생성하고 실행 명령어에 `uvicorn server:app --host 0.0.0.0 --port $PORT`를 입력합니다.
   - Render는 자체적으로 환경 설정을 통해 `Chrome` 브라우저를 다운로드 받고 셀레니움을 완벽하게 돌릴 수 있는 환경을 100% 지원합니다! (`build.sh` 스크립트를 통해 크롬 설치 가능)

### STEP 2. 프론트엔드 (Vercel + Next.js) 프로젝트 세팅하기
멋진 웹 화면을 만들어 Vercel에 무료로 배포해 봅시다.

1. **새 폴더 생성 및 Next.js 설치**:
   터미널을 열고 다음 명령어를 칩니다.
   ```bash
   npx create-next-app@latest frontend-web
   cd frontend-web
   ```
   (설치를 묻는 창이 나오면 모두 기본값-Enter-으로 넘어가시면 됩니다.)

2. **화면 디자인 만들기**:
   `app/page.tsx` 파일을 열고 리액트(Next.js) 코드를 작성합니다.
   * 입력 칸: `몇 분 주기로 예약 확인을 시작할까요? [ 12 ] 분`
   * 버튼: `[ 예약 스크립트 가동하기 ]`

3. **API 통신 연결**:
   방금 버튼을 눌렀을 때, STEP 1에서 만든 Render 웹 서버 주소(예: `https://tennis-bot.onrender.com/start`)로 `12`라는 숫자를 전송(POST 요청)하도록 자바스크립트 코드를 연결합니다.

### STEP 3. Vercel(버셀)에 배포하여 최종 도메인 얻기
1. [Vercel.com](https://vercel.com/) 에 접속합니다.
2. Vercel에서 `Add New Project`를 눌러 `frontend-web` 폴더가 들어있는 GitHub 저장소를 선택합니다.
3. 클릭 두 번 만에 `https://my-tennis-web.vercel.app` 이라는 멋진 무료 웹사이트가 생성됩니다!

---

## 🎯 전체 흐름 (작동 방식 요약)
1. 핸드폰으로 `내도메인.vercel.app` 접속
2. 화면에 `12` 입력 후 [시작] 버튼 터치
3. Vercel(리치 UI) ➡️ Render(파이썬 API) 로 신호 전송
4. Render 서버 내부에서 셀레니움 크롬 창을 안 보이게 띄우고 `main.py` 크롤러를 12분 간격으로 영구 구동
5. 테니스장 빈자리 발견 시 내 핸드폰(텔레그램)에 알림 & 자동 예약!
