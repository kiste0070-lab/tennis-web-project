# 🎾 테니스 예약 웹 서비스 (로컬 서버 + Vercel)

Render.com의 불안정性问题으로 데스크탑 로컬 서버로 연결하는 새로운 아키텍처입니다.

---

## 🛠️ 아키텍처 개요

```
[Vercel 프론트엔드] ──→ [ngrok Tunnel] ──→ [내 데스크탑 FastAPI 서버] ──→ [Selenium 봇]
```

| 구성요소 | 설명 |
|---------|------|
| **프론트엔드** | Vercel + Next.js (사용자 인터페이스) |
| **백엔드** | 내 데스크탑에서 실행하는 FastAPI 서버 |
| **연결** | ngrok으로 로컬 서버를 인터넷에 노출 |
| **실행** | Selenium 크롬 드라이버로 테니스장 예약 |

---

## 📋 준비물

1. **Node.js** - Vercel 배포용 (https://nodejs.org)
2. **Python 3.8+** - 로컬 서버 실행용
3. **Google Chrome** - Selenium용
4. **ngrok** - 로컬 서버 인터넷 노출용 (https://ngrok.com)
5. **GitHub 계정** - Vercel 연동용

---

## 🚀 Step 1: 프로젝트 클론

GitHub에서 프로젝트를 클론합니다:

```bash
git clone https://github.com/your-username/tennis-web-project.git
cd tennis-web-project
```

또는 기존에 가지고 계시면 이 단계는 건너뜁니다.

---

## 🚀 Step 2: 로컬 서버 설정

### 2-1. Python 가상환경 생성 (권장)

```bash
cd tennis-web-project/backend
python -m venv venv

# Windows의 경우
venv\Scripts\activate

# Mac/Linux의 경우
source venv/bin/activate
```

### 2-2. 의존성 설치

```bash
pip install fastapi uvicorn selenium webdriver-manager requests python-dotenv pydantic
```

### 2-3. .env 파일 설정

`backend/.env` 파일이 없다면 생성합니다:

```env
TELEGRAM_TOKEN=your_telegram_token
TELEGRAM_USER_ID=your_telegram_user_id
```

---

## 🚀 Step 3: 로컬 서버 실행

### 3-1. 서버 시작

```bash
cd tennis-web-project/backend
uvicorn server:app --host 0.0.0.0 --port 8000
```

**출력:**
```
INFO:     Started server process [1234]
INFO:     Waiting for application startup.
INFO:     Application startup complete.
INFO:     Uvicorn running on http://0.0.0.0:8000
```

### 3-2. 동작 확인

브라우저에서 `http://localhost:8000` 접속能看到:
```json
{"msg": "✅ 백엔드 봇 서버가 정상적으로 동작 중입니다!"}
```

---

## 🚀 Step 4: ngrok 설정

### 4-1. ngrok 설치 (첫 번째만)

https://ngrok.com/download 에서 다운로드하고 압축 해제합니다.

### 4-2. 인증 설정 (첫 번째만)

```bash
# Windows
ngrok.exe authtoken your_authtoken

# Mac
./ngrok authtoken your_authtoken
```

(ngrok 대시보드에서 토큰을 확인할 수 있습니다)

### 4-3. 터널 실행

새로운 터미널을 열고:

```bash
ngrok http 8000
```

**출력:**
```
Session Status                online
Account                       your-email@example.com
Region                        United States (us)
Forwarding                    https://abc123defgh.ngrok-free.app -> http://localhost:8000
```

### 4-4. ngrok 주소 확인

화면에 표시되는 `https://xxx.ngrok-free.app` 주소를 복사합니다.

**⚠️ 주의:** ngrok 터널은 재시작할 때마다 주소가 바뀝니다!

---

## 🚀 Step 5: Vercel 프론트엔드 배포

### 5-1. Vercel에 로그인

https://vercel.com 에서 GitHub 계정으로 로그인합니다.

### 5-2. 프로젝트 가져오기

1. `Add New...` → `Project` 클릭
2. GitHub 저장소 선택
3. `frontend` 폴더가 인식되는지 확인
4. `Deploy` 클릭

### 5-3. 도메인 확인

배포 완료 후 `*.vercel.app` 도메인을 얻습니다.

---

## 🚀 Step 6: 프론트엔드 설정

### 6-1. 웹사이트 접속

브라우저에서 Vercel 도메인을 엽니다.

### 6-2. API URL 입력

화면에:
1. **🔗 백엔드 API URL** 입력란에 ngrok 주소 붙여넣기
   - 예: `https://abc123defgh.ngrok-free.app`
2. [저장] 버튼 클릭

### 6-3. 실행 주기 설정

1. **실행 주기 (분)**에 숫자 입력 (예: 12)
2. [▶ 시작하기] 클릭

### 6-4. 로그 확인

[📋 로그 보기] 버튼을 클릭하면 실시간 로그가 표시됩니다.

## 🛠️ 트러블슈팅

### Q: "서버에 연결할 수 없습니다"
**A:** 
- 로컬 서버가 실행 중인지 확인
- ngrok이 실행 중인지 확인
- ngrok 주소가 올바른지 확인

### Q: 크롬 드라이버 오류
```
selenium.common.exceptions.WebDriverException
```
**A:**
```bash
pip install --upgrade webdriver-manager
```
또는 Chrome 버전에 맞는 chromedriver를 수동 설치

### Q: ngrok 주소가 자꾸 바뀜
**A:** ngrok 유료 플랜(paid plan)을 사용하면 고정 도메인을 얻을 수 있습니다.

### Q: 텔레그램 알림이 오지 않음
**A:** `.env` 파일의 `TELEGRAM_TOKEN`과 `TELEGRAM_USER_ID`가 올바른지 확인

### Q: CORS 오류
```
Access to fetch has been blocked by CORS policy
```
**A:** server.py에서 `allow_origins=["*"]`로 설정되어 있는지 확인
